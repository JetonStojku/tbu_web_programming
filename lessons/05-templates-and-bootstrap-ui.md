# Lesson 05: Templates and Bootstrap UI

## Lesson Goal

Understand how Django templates render dynamic content and how Bootstrap is used for quick UI structure.

## What Students Will Build/Learn

- Use template inheritance with a shared base layout.
- Apply key template tags and filters used in this project.
- Handle authenticated/anonymous conditional rendering.
- Use CSRF tokens for safe POST forms.

## Project Files Covered

- `blog/templates/blog/base.html`
- `blog/templates/blog/home.html`
- `blog/templates/blog/post_detail.html`
- `blog/templates/blog/post_form.html`
- `blog/templates/user/login.html`

## Step-by-Step Explanation

1. Open `blog/templates/blog/base.html`:
   - common page shell
   - Bootstrap 5.3.0 CDN inclusion (CSS + JS bundle)
   - navigation links
   - `{% block content %}` placeholder.
2. Inspect inheritance in child templates:
   - `{% extends "blog/base.html" %}` in list/detail/form/auth pages.
3. Review template tags/filters currently used:
   - tags: `{% url %}`, `{% if %}`, `{% for %}`, `{% empty %}`
   - filters: `truncatechars`, `date`, `timesince`, `linebreaks`.
4. Review authentication-aware rendering:
   - show/hide links based on `user.is_authenticated`.
5. Review CSRF usage:
   - all POST forms include `{% csrf_token %}`.

```powershell
python manage.py runserver
```

Render checks:

- Home list rendering from `posts` context.
- Post detail rendering including comments.
- Form rendering for create/update pages.
- Confirm Bootstrap assets are loaded from `blog/templates/blog/base.html`:
  - `bootstrap.min.css` (5.3.0)
  - `bootstrap.bundle.min.js` (5.3.0)

## Common Mistakes

- Hardcoding URLs instead of using `{% url %}`.
- Forgetting `{% csrf_token %}` in POST forms.
- Putting business logic in templates instead of views/forms.
- Missing `{% extends %}` and duplicating layout code in every page.

## Exercise

1. Improve one UI section in `post_detail.html` or `home.html`.
2. Use at least one existing filter (for example `date` or `truncatechars`) to improve readability.
3. Keep all navigation links generated by `{% url %}`.

## Expected Result

- Students can explain how context data appears in templates.
- Students can safely handle POST forms with CSRF protection.

### Quick Recap

Templates are for presentation logic. Inheritance and built-in tags/filters keep pages consistent and maintainable.
